### Vite config
Also Claude:

Vite runs with the React plugin for JSX/dev refresh, then `vite-plugin-compression` emits both gzip and Brotli versions of every file so the server can pick whichever the browser prefers. Production builds get the kitchen-sink treatment: Terser strips all logging, runs multiple aggressive (and slightly unsafe) passes, mangles top-level names and properties that start with `_`, and writes ASCII-only output without comments. Rollup is set to keep everything in one chunk, compact formatting, and only emit modern JS constructs. CSS goes through Lightning CSS, sourcemaps are disabled, and anything under ~100 KB gets inlined as base64. TL;DR the config is obsessed with squeezing every last byte for serving tiny bundles.
